# TOKINDLE — iOS 快捷指令配置（Phase 3）

用「快捷指令」把微信文章链接发给 TOKINDLE 后端，生成 EPUB 并保存到运行后端的电脑上的 `output/`。

> **详细英文步骤**：见 **`docs/iOS_SHORTCUT_SETUP.md`**（Step-by-step in English）。

---

## 前提

- 运行后端的电脑与 iPhone **在同一 Wi‑Fi 局域网**，且后端已按下方「局域网启动方式」启动。
- 若需要在外网或非同一 Wi‑Fi 下使用，可选用 **内网穿透**（见文末「可选：内网穿透」）。

---

## 局域网启动方式（推荐）

在**运行后端的 Mac** 上：

1. 终端执行（二选一）：
   ```bash
   cd /path/to/tokindle
   uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```
   或直接运行项目根目录脚本：`./run_lan.sh`（会输出本机 IP 供手机填写）。  
   `--host 0.0.0.0` 表示监听所有网卡，手机才能通过局域网 IP 访问。

2. 查看本机 IP（二选一）：
   - 系统设置 → 网络 → Wi‑Fi → 详情 → IP 地址
   - 或终端执行：`ipconfig getifaddr en0`（常见为 `192.168.x.x`）

3. 在手机浏览器访问：`http://你的Mac的IP:8000/ping`  
   若返回 `{"ping":"pong"}`，说明手机已能访问后端。

---

## 步骤 1：新建快捷指令

1. 打开 **「快捷指令」**（Shortcuts）。
2. 右上角 **+** 新建快捷指令。
3. 命名：例如 **「TOKINDLE 转 EPUB」**。

---

## 步骤 2：获取链接

任选一种方式：

- **从剪贴板**：添加动作 **「获取剪贴板」** → 再 **「设定变量」**，变量名 `链接`。
- **手动输入**：添加动作 **「询问输入」**，提示写「粘贴微信文章链接」，输入类型选「文本」→ **「设定变量」**，变量名 `链接`。

---

## 步骤 3：调用后端 API

1. 添加动作 **「获取 URL 内容」**（Get Contents of URL）。
2. 在 **「URL」** 中填写（把 `192.168.x.x` 换成你 Mac 的局域网 IP）：
   - **局域网**：`http://192.168.x.x:8000/parse-url`
3. 展开 **「显示更多」**（或「高级」）：
   - **方法**：**POST**
   - **请求头**：  
     添加一条：**名称** `Content-Type`，**值** `application/json`
   - **请求体**：**JSON**
     - 内容填写（把 `链接` 作为变量插入）：
       ```json
       {"url": "链接"}
       ```
     - 在 Shortcuts 里通常用「字典」或「文本」构造：键 `url`，值选变量「链接」。若用「文本」则写：`{"url":"<链接变量>"}`，并在占位符处插入变量「链接」。

4. 将 **「URL 内容」**（即请求返回）**设定为变量**，例如变量名 `响应`。

---

## 步骤 4：显示结果

1. 添加动作 **「显示结果」**（或「显示通知」/「显示提醒」）。
2. 内容选择变量 **「响应」**，这样会看到后端返回的 JSON（含 `success`、`path`、`title` 等）。

可选：再加 **「如果」** 判断「响应」包含 `"success":true`，成立时显示「已保存到服务器」，否则显示「失败」和响应内容。

---

## 步骤 5：使用方式

1. 在微信中打开公众号文章，点右上角 **「…」** → **「复制链接」**。
2. 运行快捷指令 **「TOKINDLE 转 EPUB」**（若用「询问输入」则粘贴链接；若用剪贴板则直接运行）。
3. 查看 **「显示结果」** 中的返回信息，确认是否成功及服务器上的保存路径。

---

## 后端返回示例

成功时大致为：

```json
{"success": true, "path": "/abs/path/to/output/文章标题.epub", "title": "文章标题"}
```

失败时会有 `detail` 等字段，可在快捷指令里把「响应」原样显示便于排查。

---

## 可选：内网穿透（外网或非同一 Wi‑Fi 时）

若希望手机在**其他网络**下也能访问（例如不在家时），可使用内网穿透工具，把本机 8000 端口暴露为公网 HTTPS 地址，例如：

- **ngrok**：`ngrok http 8000`，得到 `https://xxxx.ngrok-free.app`，快捷指令 URL 填 `https://xxxx.ngrok-free.app/parse-url`（需注册并配置 authtoken，见 ngrok 文档）。
- 其他类似工具（如 **frp**、**cloudflared** 等）按各自文档配置即可。

此时后端仍可只监听本机（`uvicorn main:app --reload`），由穿透工具转发到 8000；快捷指令中的 URL 改为穿透得到的公网地址 + `/parse-url`。  
**日常同一 Wi‑Fi 下使用无需内网穿透，局域网方案即可。**
